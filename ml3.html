Getting Rich Quick with Horse Racing - Part 3: Code


Step 1 - Find the Best Model using the ML.NET AutoML API


Walkthrough
  • Open Visual Studio and create a new .NET 6 Core console app (C#)
  • Install the Microsoft.ML.AutoML nuget package
  • Add using statements:
    •                 
    using Microsoft.ML;
    using Microsoft.ML.AutoML;   
    using Microsoft.ML.Data;
  • Create classes for you inputs and outputs:

    Notice I have set the name of coloumn 0 to 'Label' - this will be the ground truth used for training - it is true/false for winner/loser

    • 
            #region model input class
            public class ModelInput
            {
                [LoadColumn(0)]
                [ColumnName(@"Label")]
                public bool Col0 { get; set; }
            
                [LoadColumn(1)]
                [ColumnName(@"col1")]
                public float Col1 { get; set; }
                [LoadColumn(2)]
                [ColumnName(@"col2")]
                public string Col2 { get; set; }
                [LoadColumn(3)]
                [ColumnName(@"col3")]
                public string Col3 { get; set; }
                [LoadColumn(4)]
                [ColumnName(@"col4")]
                public float Col4 { get; set; }
                [LoadColumn(5)]
                [ColumnName(@"col5")]
                public float Col5 { get; set; }           
                [LoadColumn(6)]
                [ColumnName(@"col6")]
                public float Col6 { get; set; }
            
            }
            
            #endregion
                                  
            public class ModelOutput
            {
                [ColumnName(@"col0")]
                public bool Col0 { get; set; }
            
                [ColumnName(@"col1")]
                public float Col1 { get; set; }
                       
                [ColumnName(@"Features")]
                public float[] Features { get; set; }
            
                [ColumnName(@"PredictedLabel")]
                public bool PredictedLabel { get; set; }
            
                [ColumnName(@"Score")]
                public float Score { get; set; }
            
                [ColumnName(@"Probability")]
                public float Probability { get; set; }
            
            }
  • Create the ML Context:

    You only instatiate this once and it is used throughout the process of data preperatation, training, evaluatation and prediction.

    • 
    MLContext mlContext = new MLContext();
  • Load the data into an IDataView

    If your data has no header with column names, then specificy 'hasHeader: false:', coloumn names col0, col1, col2... will be automatically assigned.

    • 
    IDataView testDataView = mlContext.Data.LoadFromTextFile(  
                path: @"d:\data\horsedata.tsv",
                hasHeader: false,
                separatorChar: '\t',
                allowQuoting: true,
                allowSparse: false
                );
  • Create a preprocessing pipeline.

    Here you will do any transforms required and then concatenate your columns into one feature vector.

    Our numeric columns have already been normalised, so we don't need to transform them. Column 3 is text (history of recent races), so we need to apply FeaturizeText. It will apply a range of typical text transforms requried for ML (Text needs to be transformed into a vector of type float) - see the documentation

    
    var preprocessPipeline = mlContext.Transforms.Text.FeaturizeText(inputColumnName: @"col3", outputColumnName: @"Col3")      
                               .Append(mlContext.Transforms.Concatenate("Features", new[] { "Col3", "col4", "col5", "col6" }));
  • Create an ITransformer by passing the data to our preprocessing pipline

    Then use this transomer to transform the data to a new IDataView

    
    ITransformer dataPrep = preprocessPipeline.Fit(testDataView);   
    IDataView dataTransformed = dataPrep.Transform(testDataView);
  • Split the transformed data into train and test sets: 
    
    DataOperationsCatalog.TrainTestData datasplit = mlContext.Data.TrainTestSplit(dataTransformed, 0.15, seed: 1);      
    IDataView traindata = datasplit.TrainSet;
    IDataView testdata = datasplit.TestSet;
  • Run a Binary CLassification Experiment to find the best model

    • The default optimising metric is Accuracy - more on that later

    
    ExperimentResult experimentResult = mlContext.Auto()
            .CreateBinaryClassificationExperiment(setting).Execute(trainData: traindata, validationData: testdata,  
                         labelColumnName: "Label", preFeaturizer: preprocessPipeline, progressHandler: null);
  • Display the Results:

    ConsoleHelper is in the common namespance and can be found in the machinelearning-samples in the dotnet GitHub

    
    RunDetail bestrun = experimentResult.BestRun;
    BinaryClassificationMetrics mets = bestrun.ValidationMetrics;
    ConfusionMatrix mtrx = mets.ConfusionMatrix;
    Console.WriteLine("RESULTS FROM EXPERIMENT:");
    Console.WriteLine("Optimizing metric: " + setting.OptimizingMetric.ToString());
    ConsoleHelper.PrintBinaryClassificationMetrics(bestrun.TrainerName, mets);                        
    Console.WriteLine(mtrx.GetFormattedConfusionTable());

Interpret the Results:

Are we going to get rich?

  1. Accuracy is 92% - GREAT

  2. BUT - Positive Precision (Positive Predictive Value) is barely above chance at 54%

    The algorithm only picked 13 winners, and of those it was only correct for 7. So we would have bet on 13 horses and only won on 7 of them.

    Why is Positive Precision so Low while Accuracy is High?

    - The TEST POSITIVE RATIO (Incidence) is only 7.83%, (there were 600 losers to 51 winners).

    - NEGATIVE RECALL (SPECIFICITY) was great at 99%, but this still means 1% of 600 (6) were predicted as winners when they were not.

    - POSITIVE RECALL (SENSITIVITY) was only 13.7% meaning only 7 of the 51 winners were predicted correctly.

    We need to work on improving the Positive Recall and/or Postitive Precision

  3. We don't know if this will generalise to new races.

    i.e. we don't yet know how much chance was involved in the result. We could have got either lucky or unlucky with our test train split

In Part 4 we will try training to target different optimizing metrics and employing cross-validation to see how generalisable our model is.
